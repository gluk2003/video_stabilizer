# Стабилизация видео с использованием оптического потока

## 📋 Описание проекта

Этот проект реализует полный пайплайн для стабилизации видео с заметной дрожью камеры (телефонная съёмка, action-камера, GoPro, записанная вручную панорама). Используются методы компьютерного зрения на основе оптического потока для анализа и компенсации нежелательных движений камеры.

## 🎯 Основные возможности

- **Вычисление оптического потока**: поддержка плотного (Farnebäck) и разреженного (Lucas-Kanade) методов
- **Оценка глобального движения**: аффинная модель и гомография
- **Сглаживание траектории**: скользящее окно, фильтр Савицкого-Голея, фильтр Калмана
- **Компенсация движения**: warp каждого кадра к стабилизированной траектории
- **Детальный анализ ошибок**: визуализация проблемных участков и их причин
- **Визуализации**: сравнение до/после, графики траекторий, примеры оптического потока

## 📊 Методология

### 1. Вычисление оптического потока

#### Метод Farnebäck (плотный поток)

**Преимущества:**
- Плотное покрытие всего изображения
- Устойчивость к шуму
- Хорошо работает для глобального движения

**Недостатки:**
- Медленнее вычисляется
- Менее точен для резких движений

**Параметры:**
```python
farneback_params = {
    'pyr_scale': 0.5,      # Коэффициент масштабирования пирамиды
    'levels': 3,           # Количество уровней пирамиды
    'winsize': 15,         # Размер окна усреднения
    'iterations': 3,       # Итераций на каждом уровне
    'poly_n': 5,           # Размер окна полинома
    'poly_sigma': 1.2,     # Стандартное отклонение Гаусса
}
```

#### Метод Lucas-Kanade (разреженный поток)

**Преимущества:**
- Быстрее работает
- Точнее для углов и границ
- Меньше использует памяти

**Недостатки:**
- Может терять точки отслеживания
- Требует наличия хороших особых точек

**Параметры:**
```python
lk_params = {
    'maxCorners': 200,        # Максимум точек для отслеживания
    'qualityLevel': 0.01,     # Минимальное качество угла
    'minDistance': 30,        # Минимальное расстояние между точками
    'blockSize': 3,           # Размер блока для детектора
}
```

### 2. Модели глобального движения

#### Аффинная трансформация (6 параметров)
- Сдвиг по X, Y
- Поворот
- Масштабирование
- Сдвиг (shear)

**Когда использовать:** Большинство случаев съемки с рук, когда камера остается примерно на одном расстоянии от сцены.

#### Гомография (8 параметров)
- Все параметры аффинной трансформации
- Дополнительно учитывает перспективу

**Когда использовать:** Движение камеры вперед/назад, съемка с дрона, сложные 3D движения.

### 3. Методы сглаживания траектории

#### Скользящее среднее (Moving Average)

**Формула:**
```
smoothed[i] = mean(trajectory[i-r:i+r])
```

**Когда использовать:**
- Равномерная дрожь
- Простая обработка
- Предсказуемое поведение

#### Фильтр Савицкого-Голея (Savitzky-Golay)

**Особенности:**
- Полиномиальное сглаживание
- Сохраняет пики и тренды
- Хорош для динамичного видео

#### Фильтр Калмана (Kalman Filter)

**Особенности:**
- Адаптивное сглаживание
- Оптимален при известной модели шума
- Учитывает скорость движения

### 4. Компенсация движения

Для каждого кадра вычисляется коррекция:

```
correction[i] = smoothed_trajectory[i] - original_trajectory[i]
```

Затем применяется обратная аффинная/гомография трансформация.

### Метрики качества

**Снижение вариации движения:**
```
reduction = (1 - std(smoothed) / std(original)) * 100%
```

Типичные значения:
- 70-90% для хорошей стабилизации
- 50-70% для умеренной стабилизации
- <50% недостаточная стабилизация

## ⚠️ Типичные проблемы и решения

### 1. Motion Blur (размытие движения)

**Признаки:**
- Низкое качество оптического потока
- Много потерянных точек отслеживания
- Большие ошибки оценки

**Решения:**
- Увеличить `levels` в параметрах Farnebäck
- Использовать пирамидальную обработку
- Предобработка: деблюр, повышение резкости

### 2. Недостаток текстуры

**Признаки:**
- Мало детектированных точек (Lucas-Kanade)
- Ошибочная оценка движения на однородных участках
- Проблемы в кадрах "Нет точек для отслеживания"

**Решения:**
- Переключиться на Farnebäck
- Увеличить `maxCorners`
- Уменьшить `qualityLevel`
- Использовать контекст соседних кадров

### 3. Rolling Shutter

**Признаки:**
- Искажение вертикальных линий
- "Желе-эффект" при быстром движении
- Разная трансформация для разных частей кадра

**Решения:**
- Специализированные алгоритмы коррекции rolling shutter
- Использование глобального shutter при съемке
- Частичная компенсация через локальные трансформации

### 4. Окклюзии и движущиеся объекты

**Признаки:**
- Резкие скачки в траектории
- Неправильная оценка при появлении крупных объектов
- Ошибки в error analysis

**Решения:**
- Использовать RANSAC для робастной оценки
- Сегментация переднего плана
- Маскирование движущихся объектов
- Увеличить `maxIters` в RANSAC

### 5. Параллакс (3D эффекты)

**Признаки:**
- Разное движение для близких и дальних объектов
- Невозможность подобрать одну глобальную трансформацию
- Артефакты на границах объектов

**Решения:**
- Переключиться на гомографию
- Использовать depth-aware стабилизацию
- Сегментация на слои глубины

### 6. Резкие легитимные движения

**Признаки:**
- Сглаживание теряет важные повороты камеры
- Видео выглядит "заторможенным"
- Потеря динамики

**Решения:**
- Уменьшить `smoothing_radius`
- Использовать Савицкий-Голей вместо скользящего среднего
- Адаптивное сглаживание (детекция преднамеренных движений)

## 📚 Теоретические основы

### Оптический поток

Оптический поток описывает видимое движение точек в последовательности изображений, основываясь на предположениях:

1. **Brightness constancy** - яркость точки сохраняется между кадрами
2. **Temporal persistence** - малые смещения между кадрами
3. **Spatial coherence** - соседние пиксели движутся похожим образом

### Метод Farnebäck

Основан на полиномиальной аппроксимации окрестности пикселя. Использует пирамидальный подход для обработки больших смещений.

### Метод Lucas-Kanade

Минимизирует сумму квадратов разностей между окнами в двух изображениях. Решается система линейных уравнений для каждого окна.

## 🔍 Error Analysis

Notebook автоматически выполняет анализ ошибок и выявляет:

1. **Кадры с недостаточным количеством точек отслеживания**
   - Указывает на области с низкой текстурой
   - Может быть вызвано motion blur или окклюзиями

2. **Участки с большими смещениями**
   - Могут указывать на резкие движения камеры
   - Требуют более агрессивного сглаживания

3. **Проблемы с оценкой трансформации**
   - RANSAC не нашел достаточно inliers
   - Возможно наличие движущихся объектов или параллакса

4. **Частотный анализ**
   - FFT показывает какие частоты дрожи преобладают
   - Помогает выбрать оптимальный радиус сглаживания

## 📝 Основные наблюдения

### Что хорошо работает:

1. **Farnebäck + Affine + Moving Average**
   - Универсальное решение для большинства случаев
   - Стабильное и предсказуемое поведение
   - Хорошее соотношение скорость/качество

2. **Lucas-Kanade для быстрых сцен**
   - Значительно быстрее
   - Точнее при наличии хороших углов
   - Меньше использует памяти

3. **Калман для адаптивности**
   - Лучшие результаты при предсказуемых паттернах
   - Адаптируется к изменениям в характере движения
   - Оптимален для профессиональной съемки

### Ограничения:

1. **Глобальная модель не учитывает локальные искажения**
   - Rolling shutter требует специализированной обработки
   - Параллакс нарушает предположение о плоской сцене

2. **Обрезка границ уменьшает поле зрения**
   - Необходим компромисс между стабильностью и FOV
   - Альтернатива: content-aware fill границ

3. **Вычислительная сложность**
   - Real-time обработка требует оптимизации
   - GPU ускорение существенно помогает
